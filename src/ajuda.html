Simulador de Paginação de Memória Virtual

Manual do Usuário: Entendendo o Sistema

1. O que é este simulador?

Este simulador foi criado como uma ferramenta prática e interativa para tornar compreensível o conceito de Gerenciamento de Memória, com foco total na técnica de Paginação.

No funcionamento de um computador, o sistema operacional precisa organizar os dados na memória RAM de forma extremamente eficiente. Frequentemente, um programa não encontra um espaço grande o suficiente e contínuo para ser carregado. A paginação resolve esse gargalo dividindo o processo em pequenos "pedaços" uniformes (as páginas), que podem ser alocados em locais espalhados pela memória física (os quadros).

Este sistema atua exatamente como uma MMU (Memory Management Unit). Ele realiza a ponte "invisível" entre o endereço que o software solicita e o endereço físico onde o dado realmente reside. O objetivo é permitir que você visualize, passo a passo, como essa tradução ocorre e como o sistema lida com a organização da memória.

2. Guia Detalhado: Como utilizar o simulador

Não se preocupe se o conceito parece complexo. Siga estes passos para realizar sua primeira simulação:

  1. Defina o Cenário (Configurações): No primeiro card, você verá três campos. O sistema trabalha com "blocos". Digite valores que sejam potências de 2 (como 16, 32, 64, 128). Exemplo: Se definir 16 em "Tamanho da Página", você está configurando que cada "caixa" de memória comporta exatamente 16 unidades de informação. Clique em "Aplicar e Reiniciar" para o sistema calcular a arquitetura.
  2. Construa sua Tabela (Mapeamento): Imagine que a "Página Lógica" é o que o seu programa acha que existe, e o "Quadro Físico" é onde o dado realmente mora na memória RAM. Na tabela, digite números no campo "Quadro Físico" para indicar para qual lugar da memória cada página deve ir. Dica: Se você deixar um campo em branco, o simulador irá simular um "Page Fault" (Falta de Página), que é o sinal de que o dado não está na memória física e o sistema precisaria buscá-lo no disco.
  3. A "Mágica" da Tradução: No campo "Endereço Lógico Decimal", digite um número (ex: 21). Ao clicar em "Traduzir", o simulador fará o trabalho pesado: ele mostrará como o endereço é convertido em binário, separando os bits que identificam a página dos bits que indicam o "deslocamento" (a posição exata dentro da página).
  4. Interprete o Resultado: O sistema exibirá o cálculo final do endereço físico. Isso prova, na prática, que é possível organizar dados de forma eficiente, mesmo que eles estejam fragmentados na memória física.

3. Glossário: Entendendo os termos técnicos

Para dominar o simulador, é fundamental compreender estes conceitos:

  • Página (Page): A unidade de divisão lógica da memória do processo. É como se você dividisse um livro em capítulos de tamanho fixo.
  • Quadro (Frame): A unidade de divisão da memória física (RAM). O quadro deve ter sempre o mesmo tamanho da página para que o encaixe seja perfeito.
  • Offset (Deslocamento): É o endereço interno, ou seja, a posição do byte exato dentro de uma página ou quadro.
  • Tabela de Páginas: Funciona como um dicionário. Ela mantém o "endereço" de onde cada página lógica foi guardada na memória física.
  • Page Fault (Falta de Página): Um erro de acesso controlado que ocorre quando a página que o programa precisa não foi encontrada na RAM.
